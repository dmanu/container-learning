Docker Monitoring Lab

Welcome to the Docker Monitoring Lab! This environment demonstrates how to set up a complete monitoring solution for a containerized application using Docker, Prometheus, and Grafana.

Available Services

Web UI

NGINX web server serving static content (this page)

URL: http://localhost:8080

API

Flask API with Prometheus metrics

URL: http://localhost:5001

Endpoints:

  • / - Basic hello message
  • /api/data - Sample data endpoint
  • /api/health - Health check endpoint
  • /metrics - Prometheus metrics

Prometheus

Metrics collection and storage

URL: http://localhost:9090

Try these queries:

  • API request rate
  • Data requests counter

Grafana

Visualization and dashboards

URL: http://localhost:3000

Login: admin / admin

Pre-configured dashboard:

View API Dashboard

Generate Test Data

Run this command to generate some test data for the monitoring system:

for i in {1..50}; do 
  curl http://localhost:5001/
  curl http://localhost:5001/api/data
  curl http://localhost:5001/api/health
  sleep 0.5
done

Architecture

This lab demonstrates a complete monitoring pipeline:

  1. The API exposes metrics using the prometheus-flask-exporter library
  2. Prometheus scrapes these metrics at regular intervals
  3. Grafana visualizes the metrics in a pre-configured dashboard

All configuration is done through files, making it easy to understand and modify.
